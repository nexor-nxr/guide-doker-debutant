Chapitre 1

Introduction à Docker

Qu'est-ce que Docker ?

Docker est une plateforme open source qui permet de créer, déployer et exécuter des applications dans des containers. Un container est un environnement isolé, étanche et léger qui englobe tout le nécessaire pour qu'une application fonctionne : le code, les bibliothèques, les dépendances système et la configuration.

ℹ Info
Imaginez un container Docker comme une boîte hermétique : peu importe la machine sur laquelle vous déplacez cette boîte, l'application s'exécutera de manière strictement identique.

La différence fondamentale avec une Machine Virtuelle (VM)

Critère Machine Virtuelle (VM) Container Docker
Architecture Système complet simulé avec son propre OS (lourd) Partage le noyau de l'OS hôte (très léger)
Poids standard Plusieurs Giga-octets (Go) Quelques Méga-octets (Mo)
Vitesse de démarrage Quelques minutes Quelques secondes
Ressources (CPU/RAM) Réservées et bloquées à l'avance Allouées dynamiquement selon les besoins

Pourquoi utiliser Docker ?

  • Portabilité absolue : Fonctionne à l'identique sur Linux, Windows et macOS.
  • Isolation totale : Plus de conflits de versions entre vos applications (ex: faire tourner PHP 7 et PHP 8 en même temps).
  • Reproductibilité : Élimine le fameux problème "Mais ça marchait sur ma machine !".
  • Écosystème riche : Des milliers d'images officielles prêtes à l'emploi sur le Docker Hub.

Concepts fondamentaux

Concept Définition simple
🖼️ Image Le modèle en lecture seule ou le "plan" de construction. C'est l'état figé de l'application.
📦 Container L'instance vivante et en cours d'exécution d'une image. On peut créer plusieurs containers à partir d'un seul plan.
🏪 Docker Hub Le catalogue officiel en ligne (registry) où l'on télécharge les images pré-construites.
📁 Volume Espace de stockage externe persistant qui survit à l'arrêt ou à la suppression du container.
🌐 Réseau Espace d'isolation permettant aux containers de communiquer entre eux de manière sécurisée.
Chapitre 2

Installation

Prérequis système

Élément Exigence minimale
Système Debian 10+ / Ubuntu 20.04+ (ou dérivés comme MX Linux)
Ressources 2 Go de RAM minimum (4 Go recommandé) et 10 Go d'espace disque disponible
Droits Accès utilisateur avec privilèges sudo

Installation sur Debian / Ubuntu

Étape 1 — Mise à jour et dépendances

Terminal
sudo apt update && sudo apt upgrade -y
sudo apt install -y ca-certificates curl gnupg lsb-release

Étape 2 — Ajout de la clé GPG officielle Docker

Terminal
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

Étape 3 — Configuration du dépôt

Terminal
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update

Étape 4 — Installation du moteur Docker Engine

Terminal
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

Vérification de l'installation

Terminal — Test
sudo docker run hello-world
✓ Conseil : Utiliser Docker sans sudo (Recommandé)
Par défaut, Docker requiert les privilèges root. Pour l'exécuter avec votre utilisateur classique, tapez :
sudo usermod -aG docker $USER && newgrp docker
⚠ Attention
Déconnectez-vous puis reconnectez-vous de votre session système pour appliquer pleinement l'adhésion au groupe.
Chapitre 3

Commandes essentielles

Voici l'ensemble des commandes indispensables au quotidien pour manipuler sereinement vos environnements.

Gestion des images

Commande Description / Utilité
docker images Lister toutes les images téléchargées localement.
docker pull nginx Télécharger l'image Nginx depuis le Docker Hub sans la lancer.
docker rmi nginx Supprimer une image locale devenue inutile.
docker image prune Nettoyer les images intermédiaires ou orphelines.

Gestion des containers

Commande Description / Utilité
docker ps Lister uniquement les containers en cours d'exécution.
docker ps -a Lister tous les containers (actifs ET arrêtés).
docker stop mon_container Arrêter proprement un container actif.
docker start mon_container Relancer un container précédemment stoppé.
docker rm mon_container Supprimer définitivement un container arrêté.
docker rm -f mon_container Forcer la suppression d'un container en cours d'exécution.

Commandes de diagnostic

Commande Description / Utilité
docker logs mon_container Afficher la console et la sortie standard de l'application.
docker logs -f mon_container Suivre les flux de logs en temps réel (mode "follow").
docker stats Suivre la consommation RAM / CPU de chaque élément.
docker exec -it mon_container bash Ouvrir une session terminal interactive à l'intérieur du container.
Chapitre 4

Zoom sur la commande "docker run"

Syntaxe globale

La commande docker run instancie et démarre un container. Sa syntaxe s'organise ainsi :

Structure
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

Les options indispensables du quotidien

D'après les spécifications techniques officielles de Docker, voici les paramètres clés à maîtriser :

  • -d ou --detach : Exécute le container en tâche de fond (arrière-plan) et affiche uniquement son ID. Permet de garder la main sur votre terminal.
  • -p ou --publish : Publie un port du container sur l'hôte. Format : -p port_machine_hote:port_container. Essentiel pour rendre un serveur web accessible.
  • --name string : Assigne un nom personnalisé et intelligible au container au lieu d'un nom généré aléatoirement.
  • -e ou --env list : Initialise des variables d'environnement indispensables pour configurer l'application (ex: mots de passe, clés d'API).
  • -v ou --volume list : Monte un dossier ou un volume persistant afin de lier le stockage de l'hôte à celui du container.
  • --restart string : Définit la politique de redémarrage (ex: always pour relancer le container automatiquement si le système redémarre ou plante).
  • --rm : Supprime automatiquement le container et ses volumes anonymes dès qu'il s'arrête (idéal pour faire des tests propres).
  • -it (--interactive + --tty) : Maintient l'entrée standard ouverte et alloue un pseudo-terminal, requis pour utiliser des interpréteurs de commandes (comme bash ou sh).
Chapitre 5

Volumes & Réseaux

Les volumes Docker (persistance)

Par défaut, un container est éphémère (volatile). Si vous écrivez des fichiers dedans et que vous le supprimez, les données disparaissent. Les volumes externalisent et sécurisent vos fichiers.

Exemple pratique : Déploiement d'une base de données persistante
# 1. Création d'un volume nommé
docker volume create mysql_data

# 2. Lancement du container relié au volume
docker run -d \
  --name base_donnees \
  -e MYSQL_ROOT_PASSWORD=mon_mot_de_passe \
  -v mysql_data:/var/lib/mysql \
  mysql:8

Ici, même si le container est détruit, vos tables de données restent sauvegardées au sein du volume mysql_data.

Les réseaux Docker (communication)

Les réseaux isolent les flux. En créant un réseau personnalisé, vos containers s'identifient par leur simple nom.

Exemple d'isolation réseau
# Création d'un réseau de type bridge
docker network create reseau_interne

# Connexion d'un container au réseau
docker run -d --name serveur_web --network reseau_interne nginx
✓ Conseil
Deux containers connectés sur le même réseau privé peuvent communiquer en utilisant leurs noms respectifs comme adresse DNS (ex: http://serveur_web).
Chapitre 6

Multi-containers avec Docker Compose

Gérer plusieurs containers à coup de commandes individuelles s'avère vite laborieux. Docker Compose permet de centraliser et d'automatiser l'intégralité d'une architecture multi-tiers au sein d'un simple fichier descriptif textuel YAML.

Structure d'un fichier compose.yml

Créez un fichier nommé docker-compose.yml pour lier un serveur web à sa base de données :

docker-compose.yml
version: '3.8'

services:
  web:
    image: nginx:latest
    ports:
      - "8080:80"
    networks:
      - reseau_projet
    restart: always

  db:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: secret_password
      MYSQL_DATABASE: site_production
    volumes:
      - db_data:/var/lib/mysql
    networks:
      - reseau_projet

volumes:
  db_data:

networks:
  reseau_projet:

Commandes Compose majeures

Positionnez votre terminal dans le dossier contenant votre fichier de configuration :

  • docker compose up -d : Démarre l'ensemble des containers configurés en arrière-plan. (Télécharge les images si nécessaire).
  • docker compose ps : Affiche l'état des services liés au projet en cours.
  • docker compose logs -f : Aggrège et affiche les logs de tous les composants en simultané.
  • docker compose down : Arrête et détruit proprement les containers et les réseaux créés (les volumes restent préservés).
Chapitre 7

Portainer — Interface graphique

Pour les utilisateurs réfractaires aux lignes de commandes, Portainer offre une interface web soignée et complète afin de monitorer graphiquement votre écosystème Docker.

Installation de Portainer

Terminal — Déploiement
# Créer le volume requis
docker volume create portainer_data

# Lancer le container Portainer
docker run -d \
  -p 9000:9000 \
  --name administration_portainer \
  --restart=always \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v portainer_data:/data \
  portainer/portainer-ce:latest

Ouvrez votre navigateur et accédez à l'adresse suivante : http://localhost:9000 pour configurer votre compte administrateur.

Chapitre 8

Dépannage & Bonnes pratiques

Résolution des erreurs courantes

❌ Erreur : Cannot connect to the Docker daemon
Cause : Le service Docker n'est pas actif en arrière-plan sur votre machine.
Solution : Relancez le démon Docker via la commande : sudo service docker start.
⚠️ Erreur : Port already in use
Cause : Le port extérieur choisi est déjà occupé par un autre processus de votre hôte.
Solution : Modifiez le port hôte (à gauche du signe deux-points), par exemple : -p 8085:80.

Les 5 Règles d'Or du bon praticien

  • Règle 1 : Utilisez toujours l'option --name pour identifier facilement vos containers.
  • Règle 2 : Ne stockez aucun fichier précieux directement dans le container. Externalisez tout dans un volume.
  • Règle 3 : N'écrivez jamais de mots de passe en dur. Utilisez des variables d'environnement.
  • Règle 4 : Adoptez Docker Compose dès que votre projet nécessite plus d'un container.
  • Règle 5 : Assurez la continuité de service avec la politique --restart always.

Commandes de nettoyage utiles

Terminal — Maintenance
# Supprimer tout ce qui est inutilisé (containers arrêtés, images orphelines)
docker system prune -a --volumes

# Analyser l'espace disque consommé par Docker
docker system df
